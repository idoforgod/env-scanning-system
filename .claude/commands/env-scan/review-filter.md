# 중복 필터링 결과 검토

오늘 날짜: !`date +%Y-%m-%d`

## 검토 대상

다음 파일들을 읽어 결과를 표시하세요:

1. **필터링 결과**
   - @data/{date}/filtered/new-signals-{today}.json

2. **제거 로그**
   - @logs/duplicates-removed-{today}.log

## 표시 형식

### 1. 필터링 요약
```
총 스캔:     {N}건
중복 제거:   {N}건
신규 통과:   {N}건
업데이트:    {N}건
```

### 2. 신규 신호 목록 (상위 20개)
| # | 제목 | 카테고리 | 출처 |
|---|------|----------|------|
| 1 | ... | ... | ... |

### 3. 제거된 신호 (검토 필요)
다음 항목들이 중복으로 제거되었습니다:

**URL 일치로 제거:**
- [제목] - 기존 신호 ID: SIG-xxx

**제목 유사도 (≥90%)로 제거:**
- [제목] (92% 유사) - 기존: SIG-xxx

**내용 유사도 (≥85%)로 제거:**
- [제목] (87% 유사) - 기존: SIG-xxx

**엔티티 중복 (플래그됨):**
- [제목] - 공유 엔티티: [엔티티들]

## 사용자 액션

- **모두 승인**: Phase 2로 진행
- **강제 포함**: `/force-include [raw_id]` 명령 사용
- **재필터링**: 기준 조정 후 `@dedup-filter` 재실행

승인하시면 Phase 2 (분석 단계)를 시작합니다.
검토가 필요한 항목이 있으면 알려주세요.
