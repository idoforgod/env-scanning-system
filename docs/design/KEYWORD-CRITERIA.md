# STEEPS 키워드 선정 원칙

> **버전**: 1.0
> **작성일**: 2026-01-14
> **갱신 주기**: 월별

---

## 1. 개요

환경스캐닝 시스템의 STEEPS 키워드는 미래 변화의 조기 징후(약한 신호)를 탐지하기 위한 핵심 도구입니다. 이 문서는 키워드 선정, 평가, 갱신의 원칙을 정의합니다.

---

## 2. 선정 원칙: 균형 접근 (Balanced Approach)

세 가지 관점을 균등하게 배합하여 키워드를 선정합니다:

```
┌─────────────────────────────────────────────────────────────────┐
│                    균형 접근 모델 (1:1:1)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│    ┌───────────────┐                                             │
│    │ 미래 영향력   │ ← 장기적 변화 가능성과 파급력이 큰 주제     │
│    │    (33%)      │   예: 양자컴퓨터, 기후변화, 인구구조        │
│    └───────┬───────┘                                             │
│            │                                                     │
│    ┌───────┴───────┐                                             │
│    │ 뉴스 출현빈도 │ ← 현재 자주 언급되는 트렌드 키워드          │
│    │    (33%)      │   예: AI, 금리, 전기차, ESG                 │
│    └───────┬───────┘                                             │
│            │                                                     │
│    ┌───────┴───────┐                                             │
│    │ 약한 신호     │ ← 아직 주류가 아닌 초기 징후                │
│    │    (33%)      │   예: 뇌-컴퓨터 인터페이스, 합성생물학      │
│    └───────────────┘                                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 카테고리별 키워드 구성

### 3.1 STEEPS 6개 카테고리

| 카테고리 | 영문 | 초점 영역 |
|----------|------|-----------|
| **S**ocial | Social | 인구, 라이프스타일, 가치관, 세대, 교육, 건강, 노동 |
| **T**echnological | Technological | AI, 바이오, 양자, 로봇, 에너지, 우주, 통신 |
| **E**conomic | Economic | 거시경제, 금융, 무역, 산업, 기업, 공급망 |
| **E**nvironmental | Environmental | 기후, 에너지전환, 오염, 생태계, 자원, 재해 |
| **P**olitical | Political | 정부, 정책, 외교, 안보, 국제관계, 규제 |
| **S**piritual | Spiritual | 가치, 윤리, 문화, 종교, 정체성, 공동체 |

### 3.2 계층적 키워드 구조

각 카테고리 내에서 키워드는 3계층으로 구성됩니다:

```
Level 1: 핵심 개념 (Core Concepts)
├── 해당 카테고리의 대표적 상위 개념
├── 예: "인공지능", "기후변화", "인구"
└── 카테고리당 5-8개

Level 2: 세부 영역 (Sub-domains)
├── 핵심 개념의 하위 분야
├── 예: "머신러닝", "딥러닝", "생성AI"
└── 카테고리당 15-25개

Level 3: 구체적 트렌드/기술 (Specific Trends)
├── 특정 기술, 기업, 현상
├── 예: "GPT", "CRISPR", "SpaceX", "MZ세대"
└── 카테고리당 15-25개
```

### 3.3 언어 구성

| 언어 | 비율 | 용도 |
|------|------|------|
| 한국어 | 80% | 네이버 뉴스, 국내 매체 |
| 영어 | 20% | 글로벌 뉴스, 학술 자료 |

## 4. 포함 기준 (Inclusion Criteria)

키워드가 포함되려면 다음 중 **2개 이상**을 충족해야 합니다:

### 4.1 미래 영향력 기준

| 항목 | 설명 | 체크 |
|------|------|------|
| 장기 파급력 | 5년 이상 사회/경제에 영향을 미칠 가능성 | ☐ |
| 다중 영역 영향 | 2개 이상의 STEEPS 카테고리에 영향 | ☐ |
| 시스템 변화 | 기존 시스템의 근본적 변화를 유발할 가능성 | ☐ |

### 4.2 뉴스 출현빈도 기준

| 항목 | 설명 | 체크 |
|------|------|------|
| 월간 출현 | 주요 언론에서 월 10회 이상 언급 | ☐ |
| 증가 추세 | 최근 3개월 출현 빈도 증가 추세 | ☐ |
| 다중 소스 | 3개 이상의 다른 매체에서 다룸 | ☐ |

### 4.3 약한 신호 기준

| 항목 | 설명 | 체크 |
|------|------|------|
| 초기 단계 | 아직 주류 언론에서 자주 다루지 않음 | ☐ |
| 전문가 주목 | 학술지/전문 매체에서 언급 시작 | ☐ |
| 혁신 잠재력 | 기존 패러다임을 변화시킬 가능성 | ☐ |

---

## 5. 제외 기준 (Exclusion Criteria)

다음 중 **1개 이상**에 해당하면 제외합니다:

| 기준 | 설명 | 예시 |
|------|------|------|
| **과도하게 일반적** | 거의 모든 기사에 나타남 | "경제", "사회", "기술" |
| **시의성 상실** | 더 이상 관련성 없는 과거 트렌드 | "Y2K", "3D TV" |
| **중복** | 이미 포함된 키워드와 의미 중복 | "AI"와 "인공지능" 모두 포함은 허용 (언어별) |
| **너무 좁음** | 특정 기업/제품에만 해당 | "아이폰15 프로" |
| **단기 이벤트** | 일회성 이벤트 | "2026 대선" (장기적 "선거"는 포함) |

## 6. 효과성 측정 메트릭

### 6.1 추적 메트릭

| 메트릭 | 설명 | 목표 |
|--------|------|------|
| **수집 건수** | 키워드별 URL 수집 수 | 월 10건 이상 |
| **신호 전환율** | 수집 URL → 최종 신호 비율 | 15% 이상 |
| **우선순위 점수** | 생성된 신호의 평균 우선순위 | 6.0 이상 |
| **0건 비율** | 결과 없는 키워드 비율 | 10% 이하 |

### 6.2 성과 등급

| 등급 | 기준 | 조치 |
|------|------|------|
| **A (우수)** | 월 20건+, 전환율 20%+ | 유지 |
| **B (양호)** | 월 10-19건, 전환율 15%+ | 유지 |
| **C (보통)** | 월 5-9건, 전환율 10%+ | 모니터링 |
| **D (저조)** | 월 1-4건, 전환율 <10% | 개선 검토 |
| **F (비효과)** | 3개월 연속 0건 | 퇴출 후보 |

## 7. 갱신 정책

### 7.1 갱신 주기

| 작업 | 주기 | 담당 |
|------|------|------|
| 효과성 메트릭 집계 | 일일 (자동) | 시스템 |
| 월간 성과 리뷰 | 월 1회 | 운영자 |
| 키워드 추가/삭제 | 월 1회 | 운영자 |
| 전체 재검토 | 분기 1회 | 운영자 |

### 7.2 키워드 추가 절차

```
1. 후보 키워드 식별
   - 새로운 트렌드 발견
   - 기존 키워드 세분화 필요
   - 빈 영역 보완

2. 포함 기준 검증
   - 3개 기준 중 2개 이상 충족 확인
   - 제외 기준 해당 여부 확인

3. 시험 운영 (1개월)
   - 임시 추가 후 메트릭 수집
   - D등급 이상 달성 시 정식 추가

4. 정식 등록
   - cache_manager.py에 추가
   - 변경 이력 기록
```

### 7.3 키워드 퇴출 절차

```
1. 퇴출 후보 식별
   - F등급 3개월 연속
   - 시의성 상실 판단

2. 최종 검토
   - 약한 신호 가능성 재검토
   - 미래 영향력 재평가

3. 퇴출 실행
   - cache_manager.py에서 제거
   - 변경 이력 기록
   - 퇴출 사유 문서화
```
